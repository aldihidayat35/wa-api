Panduan Deploy ke VPS

1 Order VPS & Setup Awal

Rekomendasi Spesifikasi Minimum:
  • CPU: 1 vCore
  • RAM: 1 GB (2 GB lebih baik)
  • Storage: 20 GB SSD
  • OS: Ubuntu 22.04 LTS atau 20.04 LTS
  • Bandwidth: Unlimited atau min 1 TB/bulan

Setelah order VPS, Anda akan mendapat:

  • IP Address (contoh: 103.xxx.xxx.xxx)
  • Username: root
  • Password

2 Koneksi ke VPS via SSH

Download dan install PuTTY (Windows) atau gunakan Terminal (Mac/Linux)

# Windows: Buka PuTTY, masukkan IP dan port 22

# Mac/Linux: Gunakan terminal
ssh root@103.xxx.xxx.xxx

# Masukkan password yang diberikan provider

3 Update System & Install Dependencies 

# Update system
sudo apt update && sudo apt upgrade -y

# Install Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Verify installation
node --version
npm --version

# Install Git
sudo apt install -y git

# Install PM2 (Process Manager)
sudo npm install -g pm2

# Install Nginx (Web Server)
sudo apt install -y nginx

4 Install MySQL/MariaDB 

# Install MariaDB
sudo apt install -y mariadb-server

# Secure installation
sudo mysql_secure_installation

# Login ke MySQL
sudo mysql -u root -p

# Buat database dan user
CREATE DATABASE wa_api;
CREATE USER 'waapi_user'@'localhost' IDENTIFIED BY 'password_kuat_anda';
GRANT ALL PRIVILEGES ON wa_api.* TO 'waapi_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

# Import database structure
mysql -u waapi_user -p wa_api < /path/to/database.sql

5 Clone & Setup Aplikasi 

# Pindah ke directory web
cd /var/www

# Clone repository (atau upload via FTP)
git clone https://github.com/aldihidayat35/wa-api.git
cd wa-api

# Install dependencies
npm install

# Build TypeScript
npm run build

# Install PHP dan Apache/Nginx untuk API endpoint
sudo apt install -y php php-mysql php-cli php-curl php-json

# Copy API files ke web directory
sudo mkdir -p /var/www/html/Baileys
sudo cp -r . /var/www/html/Baileys/

6 Konfigurasi Environment 

# Buat file .env (jika diperlukan)
nano .env

# Isi dengan konfigurasi
PORT=3000
DB_HOST=localhost
DB_USER=waapi_user
DB_PASS=password_kuat_anda
DB_NAME=wa_api

# Save: Ctrl+O, Enter, Ctrl+X

7 Jalankan dengan PM2 

# Start aplikasi dengan PM2
cd /var/www/wa-api
pm2 start npm --name "wa-api" -- run web

# Atau jika sudah build:
pm2 start web-server.js --name "wa-api"

# Set PM2 startup on boot
pm2 startup
pm2 save

# Cek status
pm2 status
pm2 logs wa-api

# Commands PM2:
pm2 restart wa-api   # Restart
pm2 stop wa-api      # Stop
pm2 delete wa-api    # Delete
pm2 monit            # Monitor

8 Konfigurasi Nginx Reverse Proxy 

# Buat konfigurasi Nginx
sudo nano /etc/nginx/sites-available/waapi.demoj35.site

# Paste konfigurasi berikut:
server {
    listen 80;
    server_name waapi.demoj35.site;
    
    # Node.js Socket.IO
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
    
    # PHP API Endpoints
    location /api/ {
        root /var/www/html/Baileys;
        index index.php;
        try_files $uri $uri/ /api/index.php?$query_string;
        
        location ~ \.php$ {
            include snippets/fastcgi-php.conf;
            fastcgi_pass unix:/var/run/php/php8.1-fpm.sock;
            fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
            include fastcgi_params;
        }
    }
    
    # Static files
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        root /var/www/html/Baileys/public;
        expires 1y;
    }
}
# Enable site
sudo ln -s /etc/nginx/sites-available/waapi.demoj35.site /etc/nginx/sites-enabled/

# Test konfigurasi
sudo nginx -t

# Restart Nginx
sudo systemctl restart nginx

9 Install SSL Certificate (HTTPS) 

# Install Certbot
sudo apt install -y certbot python3-certbot-nginx

# Generate SSL certificate
sudo certbot --nginx -d waapi.demoj35.site

# Pilih option:
# 1. Email untuk notifikasi
# 2. Agree terms: Y
# 3. Redirect HTTP to HTTPS: pilih 2

# Auto renewal (sudah otomatis, cek dengan:)
sudo certbot renew --dry-run

10 Update Domain DNS

Setting DNS di RumahWeb:
  1. Login ke panel RumahWeb
  2. Masuk ke menu Domain Management
  3. Pilih domain waapi.demoj35.site
  4. Klik Manage DNS
  5. Tambahkan/Edit A Record:
    • Type: A
    • Name: @ (atau waapi)
    • Value: IP VPS Anda (103.xxx.xxx.xxx)
    • TTL: 3600
  6. Save changes
  7. Tunggu propagasi DNS (5-60 menit)

Panduan Deploy ke Cloud (Railway/Render)

Deploy ke Railway.app (Recommended)

1. Persiapan Repository
# Push code ke GitHub
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/aldihidayat35/wa-api.git
git push -u origin main
2. Buat file railway.json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": {
    "builder": "NIXPACKS"
  },
  "deploy": {
    "startCommand": "npm run web",
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 10
  }
}
3. Deploy ke Railway
  1. Buka railway.app
  2. Login dengan GitHub
  3. New Project → Deploy from GitHub repo
  4. Pilih repository wa-api
  5. Add MySQL database (dari tab Add Service)
  6. Set environment variables di Settings
  7. Deploy!
4. Custom Domain
  1. Di Railway dashboard, masuk ke Settings
  2. Klik "Generate Domain" untuk dapat domain .railway.app
  3. Atau klik "Custom Domain" untuk waapi.demoj35.site
  4. Copy CNAME record yang diberikan
  5. Tambahkan di DNS RumahWeb:
    • Type: CNAME
    • Name: waapi
    • Value: [railway-cname].railway.app

Troubleshooting 

# Cek logs error
pm2 logs wa-api --lines 50

# Cek apakah port 3000 sudah digunakan
sudo netstat -tulpn | grep 3000

# Kill process jika ada
sudo kill -9 [PID]

# Restart PM2
pm2 restart wa-api

# Cek apakah Node.js app running
pm2 status

# Cek Nginx error log
sudo tail -f /var/log/nginx/error.log

# Test Nginx config
sudo nginx -t

# Restart services
pm2 restart wa-api
sudo systemctl restart nginx

# Test MySQL connection
mysql -u waapi_user -p wa_api

# Check MySQL status
sudo systemctl status mysql

# Restart MySQL
sudo systemctl restart mysql

# Check user privileges
mysql -u root -p
SHOW GRANTS FOR 'waapi_user'@'localhost';

Pastikan Nginx config sudah support WebSocket:

proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection 'upgrade';

Cek firewall:

# Allow port 80 dan 443
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw reload
